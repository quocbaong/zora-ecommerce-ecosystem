2️⃣ Service-to-Service Communication
Trong microservice có 2 cách giao tiếp chính
1 Sync (REST) 2 Async (Event)

1️⃣ Sync Communication
Dùng khi cần response ngay.
Ví dụ:
Order → Product Order → Cart
Example
Order Service gọi Product Service
GET /products/{id}
Java example
@RestTemplate Product product = restTemplate.getForObject( "http://product-service/products/" + id, Product.class );

2️⃣ Async Communication
Dùng khi không cần response ngay.
Ví dụ:
order_created payment_success
dùng
Kafka
Example event
{  "event":"ORDER_CREATED",  "orderId":101 }

3️⃣ Khi nào dùng Sync
Dùng khi cần:
validate data check stock get user info
Ví dụ
Order → Product Order → User

4️⃣ Khi nào dùng Async
Dùng khi:
notification inventory update analytics email
Ví dụ
Order → Notification Payment → Order

5️⃣ Quy tắc rất quan trọng (90% sinh viên làm sai)
❌ Sai
Service A → Service B → Service C → Service D
Chain dài → dễ crash.

✅ Đúng
Service A    │    ▼ Kafka Event    │  ┌─┴────────────┐  ▼              ▼ Service B      Service C

6️⃣ Ví dụ đúng trong hệ thống của bạn
